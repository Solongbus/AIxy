## AIxy 是什么？

AIxy 是一款 **AI API 整合助手**，它在你的开发工具（如 Trae,Codex,CodeBuddy 等）和 AI 服务之间充当"智能中转站"。

你只需要把开发工具的 API 地址指向 AIxy，它就会自动帮你：

- **挑选最快的线路**，让 AI 响应更快
- **压缩多余内容**，帮你在不损失效果的前提下省下大笔费用
- **记录每次对话**，方便你随时回顾和查找
- **保护你的密钥**，所有数据只存在你自己的设备上

不需要任何技术背景，开箱即用。

## 核心功能

### ⚡ 节点测速 —— 一键找到最快线路

AI 服务通常有多个接入点（节点），但速度参差不齐。AIxy 内置了专业的测速工具，你只需点击"开始测速"，它会自动测试每个节点的响应速度、稳定性，并给出直观的评分。测完之后，AIxy 会自动切换到最快的节点，让每一次对话都快人一步。

### 🧠 Token 智能压缩 —— 花更少的钱，办同样的事

每次和 AI 对话都会消耗 Token（可以理解为"AI 的计量单位"），费用不菲。AIxy 搭载了自主研发的 **CATO 缓存感知压缩引擎**，能智能识别对话中重复、冗余的内容并进行精简，在完全不影响 AI 回答质量的前提下，帮你节省可观的 Token 开销。支持从"保守"到"激进"三档压缩力度，一键切换，费用立省。

### 📊 Token 账本 —— 每一分花费都清清楚楚

AIxy 会自动记录你每天、每月的 Token 使用情况：总共用了多少、输入了多少、输出了多少、缓存命中率是多少、节省了多少。所有数据一目了然地展示在界面上，让你对自己的 AI 使用成本了如指掌，再也不会"月底看账单吓一跳"。

### 💬 会话时光机 —— 对话记录，随时回溯

每次通过 AIxy 的对话都会被完整保存下来。你可以在"会话记录"标签页中按时间浏览、按关键词搜索，快速找到之前的某段对话。无论是想回顾一个解决方案，还是找回不小心关掉的对话，都能轻松实现。

### 🛡️ 智能容错 —— 断线也能用

AIxy 内置了熔断保护机制，当某个节点出现故障时会自动隔离，避免反复重试浪费时间。同时支持多节点自动重试，确保你的 AI 对话不会因为一个节点的问题而中断。故障节点恢复后会自动重新启用，全程无需你手动干预。

## 快速开始

### 使用步骤

1. **下载** —— 从 [Releases](https://github.com/Solongbus/AIxy/releases) 页面下载最新版本
2. **运行** —— 启动 AIxy，它会安静地在后台为你服务
3. **配置节点** —— 在"节点测速"标签页添加你的 API 地址和密钥
4. **开始测速** —— 点击测速按钮，AIxy 会帮你找到最快的节点
5. **修改开发工具设置** —— 把你的开发工具的 API 地址改为 AIxy 提供的本地地址
6. **开始使用** —— 一切就绪！你的每一次 AI 对话都会经过 AIxy 的智能加速和优化

## 关于 CATO 缓存感知压缩引擎

**CATO（Cache-Aware Token Optimizer）** 是 AIxy 的核心技术之一，专为 AI 代理场景设计。

它的设计灵感来源于 DeepSeek-Reasonix 的缓存优化思路，核心原则是：**在压缩 Token 的同时，不破坏 AI 服务的自动缓存机制**。

CATO 将对话内容划分为三个区域：

| 区域 | 内容 | 处理方式 |
|------|------|----------|
| 固定前缀 | 系统提示词、首轮对话 | 完全保留，不动缓存 |
| 可压缩区 | 中间的工具调用结果等 | 智能压缩 |
| 近期尾巴 | 最近几轮对话 | 完全保留，保证连贯 |

这样既省了 Token，又不破坏缓存命中率，实现"省上加省"。
